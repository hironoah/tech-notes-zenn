# 記事レビュー skill

技術記事のPRレビューを行います。ultrathinkを使用して包括的・網羅的に品質チェックを実施してください。

## 実行方法

対象の記事ファイルを読み込み、以下の観点でレビューを行ってください。

## レビュー観点

### 対象読者
- [ ] 想定読者のレベルに合った説明か（Zenn: 中級者、Qiita: 初学者）
- [ ] 前提知識が明示されているか
- [ ] 読者が再現できる手順になっているか

### 基本構成
- [ ] タイトルは内容を的確に表しているか
- [ ] 導入で「誰向けに」「何が得られるか」が明確か
- [ ] 結論・まとめがあるか

### 技術的正確性
- [ ] コードは動作するか（構文エラーがないか）
- [ ] 技術的に誤った記述がないか
- [ ] バージョン依存の記述に注意書きがあるか
- [ ] サンプルコードは実際に動作確認できる形式か

### 読みやすさ
- [ ] 論理の流れが自然か（唐突な話題転換がないか）
- [ ] 専門用語に適切な説明があるか
- [ ] 長すぎる段落がないか
- [ ] 見出しで内容が把握できるか
- [ ] 適切な箇所に図解やコード例があるか

### 文章品質
- [ ] 誤字脱字がないか
- [ ] です/ます調で統一されているか
- [ ] 一文が長すぎないか（目安: 60文字以内）
- [ ] 冗長な表現がないか

### SEO・発見性
- [ ] slugは適切か（技術名-内容の形式）
- [ ] タグ（topics）は適切か
- [ ] 検索されそうなキーワードが含まれているか
- [ ] 類似記事との差別化ポイントがあるか

### ポートフォリオとして
- [ ] 専門性・信頼性が伝わるか
- [ ] 実務経験を感じさせる具体性があるか

## 出力形式

レビュー結果は以下の形式で出力してください：

```markdown
## レビュー結果

### 総評
（記事全体の評価を1-2文で）

### 良い点
-

### 改善が必要な点
-

### 具体的な修正提案
1.
2.
3.

### チェックリスト結果
（上記の各項目について OK/NG を記載）
```
