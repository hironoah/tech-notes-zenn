# コミット skill

変更内容を確認し、適切なコミットメッセージでコミットを行います。

## 実行手順

1. `git status` で変更ファイルを確認
2. `git diff` で変更内容を確認
3. 変更内容に基づいてコミットメッセージを作成
4. ユーザーに確認後、コミットを実行

## コミットメッセージ規則

### 形式
```
{type}({scope}): {description}
```

### type
- `add`: 新規記事・ファイル追加
- `update`: 既存記事・ファイルの更新
- `fix`: 誤字修正・バグ修正
- `wip`: 作業中（下書きなど）
- `chore`: 設定ファイル・依存関係の更新
- `docs`: ドキュメント更新

### scope
- `article`: 記事関連
- `book`: 本関連
- `drafts`: 下書き関連
- `config`: 設定関連
- 省略可（複数箇所にまたがる場合）

### 例
```
add(article): Vue3 Composablesの解説記事
update(article): TypeScript入門の誤字修正
fix(article): コードブロックの言語指定を修正
wip(drafts): React Server Componentsのネタ出し
chore(config): zenn-cliのバージョン更新
docs: READMEに使い方を追記
```

## コミット粒度

### 原則
- 1コミット = 1つの論理的な変更単位
- 後から見て「何をしたか」が分かる粒度

### 分割の目安
- **記事関連**: 1記事 = 1コミット
- **設定変更**: 関連する設定変更はまとめてOK
- **複数ファイル**: 同じ目的の変更ならまとめてOK

### 分割すべきケース
- 記事追加 + 設定変更 → 別コミット
- 複数の無関係な記事 → 別コミット
- バグ修正 + 新機能 → 別コミット

### まとめてOKなケース
- 1つの記事 + その記事用の画像
- 関連する複数ファイルの設定変更
- リファクタリング（目的が1つ）

## 確認事項

コミット前に以下を確認：
- [ ] 機密情報（.env、トークン等）が含まれていないか
- [ ] 不要なファイル（node_modules等）が含まれていないか
- [ ] コミットメッセージが変更内容を適切に表しているか
- [ ] 適切な粒度に分割されているか
